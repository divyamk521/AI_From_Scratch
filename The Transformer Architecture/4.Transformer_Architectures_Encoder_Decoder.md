# Transformer Architectures: Encoder-only, Decoder-only, and Encoder-Decoder

## 1. Encoder-only Architecture (BERT-style)

### 1-Liner Definition

An **Encoder-only** architecture is a Transformer model that reads an
entire input sequence **bidirectionally**, allowing every token to
attend to every other token to build rich contextual representations.

### Short Technical Explanation

Encoder models such as **BERT** use **bidirectional self-attention**,
meaning each token can attend to both previous and future tokens.

They are commonly trained using **Masked Language Modeling (MLM)**,
where random words are hidden and the model predicts them using
surrounding context.

Encoder-only models excel at language understanding tasks but are not
designed for open-ended text generation.

### Real-World Scenarios

-   **Search Engines:** Understanding user search intent through
    semantic matching.
-   **Named Entity Recognition (NER):** Identifying names, dates,
    organizations, and locations in documents.

### Code Snippet

``` python
import numpy as np

tokens = [0, 1, 2]
seq_len = len(tokens)

encoder_attention_mask = np.ones((seq_len, seq_len))

print("Encoder Attention Mask:")
print(encoder_attention_mask)
```

## 2. Decoder-only Architecture (GPT-style)

### 1-Liner Definition

A **Decoder-only** architecture generates text one token at a time by
predicting the next token while preventing access to future tokens.

### Short Technical Explanation

Models such as **GPT** use **causal (masked) self-attention**.

Each token can attend only to itself and earlier tokens, enabling
autoregressive text generation.

This architecture is ideal for chatbots, coding assistants, and content
generation.

### Real-World Scenarios

-   AI code completion
-   Conversational AI assistants

### Code Snippet

``` python
import numpy as np

seq_len = 3

decoder_causal_mask = np.tril(np.ones((seq_len, seq_len)))

print("Decoder Causal Mask:")
print(decoder_causal_mask)
```

## 3. Encoder-Decoder Architecture (T5-style)

### 1-Liner Definition

An **Encoder-Decoder** architecture first encodes the input into
contextual representations and then generates a new output sequence
using cross-attention.

### Short Technical Explanation

The encoder builds a bidirectional representation of the input.

The decoder then performs autoregressive generation while attending to
the encoder outputs through **Cross-Attention**, making this
architecture ideal for sequence-to-sequence tasks.

### Real-World Scenarios

-   Machine translation
-   Text summarization
-   Question answering

### Code Snippet

``` python
import numpy as np

encoder_hidden_states = np.array([
    [0.1, 0.8],
    [0.9, 0.2]
])

class ConceptualEncoderDecoder:

    def cross_attention_step(
        self,
        decoder_current_state,
        encoder_states
    ):
        scores = np.dot(
            decoder_current_state,
            encoder_states.T
        )

        weights = np.exp(scores) / np.sum(np.exp(scores))

        context = np.dot(weights, encoder_states)

        return context

model = ConceptualEncoderDecoder()

decoder_state = np.array([0.5, 0.5])

context = model.cross_attention_step(
    decoder_state,
    encoder_hidden_states
)

print(context)
```
