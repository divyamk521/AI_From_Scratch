# 1. Vector Databases

## 1-Liner Definition

A **vector database** is a specialized database that stores, indexes,
and performs extremely fast similarity searches over high-dimensional
embedding vectors.

------------------------------------------------------------------------

## Why Traditional Databases Struggle

Relational databases (MySQL, PostgreSQL) are optimized for exact matches
using B-Tree indexes.

Embedding vectors contain hundreds or thousands of floating-point
values. Without a vector index, every query requires comparing against
every stored vector (linear scan), which becomes prohibitively slow at
scale.

------------------------------------------------------------------------

## Core Architecture

``` text
Embedding
    │
    ▼
Vector Index (HNSW / IVF)
    │
    ▼
Approximate Nearest Neighbor Search
    │
    ▼
Top-K Similar Results
```

### Key Components

-   **HNSW:** Graph-based ANN index for high-speed search.
-   **IVF:** Cluster-based partitioning that searches only nearby
    clusters.
-   **Quantization:** Compresses vectors to reduce storage.
-   **Hybrid Filtering:** Combines metadata filters with vector search.

## Python Example

``` python
import math

def euclidean(a, b):
    return math.sqrt(sum((x-y)**2 for x, y in zip(a, b)))

query = [0.84, 0.11, 0.89, 0.05]
doc = [0.85, 0.12, 0.90, 0.04]

print(euclidean(query, doc))
```
