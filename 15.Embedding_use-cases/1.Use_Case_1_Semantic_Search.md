# Use Case 1: Semantic Search

## 1-Liner Definition

**Semantic search** retrieves information based on meaning instead of
exact keyword matching by using embedding vectors.

------------------------------------------------------------------------

## Architectural Overview

Semantic search converts both queries and documents into vectors and
compares them in a shared vector space.

``` text
User Query
    │
    ▼
Embedding Model
    │
    ▼
Vector Database
    │
    ▼
Cosine Similarity
    │
    ▼
Relevant Documents
```

### Process

1.  Generate an embedding for the query.
2.  Compare it with stored document embeddings.
3.  Rank results by similarity.
4.  Return the closest matches.

------------------------------------------------------------------------

## Hybrid Search

  -----------------------------------------------------------------------
  Method                  Strength                Weakness
  ----------------------- ----------------------- -----------------------
  Dense Search            Understands meaning     May miss exact IDs

  Sparse Search (BM25)    Exact keyword matching  Misses semantic intent

  Hybrid + RRF            Combines both           Slight latency increase

## Python Example

``` python
import math

def cosine_similarity(a, b):
    dot = sum(x*y for x, y in zip(a, b))
    mag1 = math.sqrt(sum(x*x for x in a))
    mag2 = math.sqrt(sum(y*y for y in b))
    return dot / (mag1 * mag2)

query = [0.89, 0.11, 0.92, 0.04]
document = [0.86, 0.14, 0.88, 0.06]

print(cosine_similarity(query, document))
```
