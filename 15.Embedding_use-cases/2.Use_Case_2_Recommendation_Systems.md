# Use Case 2: Recommendation Systems

## 1-Liner Definition

**Embedding-based recommendation systems** represent users and items as
vectors in a shared embedding space, enabling personalized
recommendations based on semantic similarity rather than manual rules.

------------------------------------------------------------------------

## Architectural Deep Dive

Modern recommendation systems typically use a **two-stage pipeline**:

``` text
Item Catalog
     │
     ▼
Retrieval Layer (Two-Tower + ANN)
     │
     ▼
~100 Candidate Items
     │
     ▼
Ranking Layer
(Context, Time, Device, Location)
     │
     ▼
Top Personalized Recommendations
```

### Two-Tower Architecture

-   **User Tower:** Encodes user history, clicks, searches, and profile
    into a **User Vector**.
-   **Item Tower:** Encodes item metadata into an **Item Vector**.
-   **ANN Search:** Finds the nearest item vectors efficiently.
-   **Ranking Model:** Re-ranks candidates using contextual features.

## Python Example

``` python
import math

def cosine_similarity(a, b):
    dot = sum(x*y for x, y in zip(a, b))
    mag1 = math.sqrt(sum(x*x for x in a))
    mag2 = math.sqrt(sum(y*y for y in b))
    return dot / (mag1 * mag2)

user = [0.02, 0.93, 0.11, 0.91]
item = [0.01, 0.91, 0.09, 0.94]

print(cosine_similarity(user, item))
```
