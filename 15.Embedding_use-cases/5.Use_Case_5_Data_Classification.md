# Use Case 5: Data Classification

## 1-Liner Definition

**Embedding-based data classification** assigns predefined labels to
unstructured text by using embedding vectors with similarity matching or
lightweight machine learning classifiers.

------------------------------------------------------------------------

## Architectural Deep Dive

Instead of relying on keyword matching, embeddings capture the semantic
meaning of text and classify it based on vector similarity or learned
decision boundaries.

``` text
Incoming Text
      │
      ▼
Embedding Model
      │
      ▼
Classification Layer
 ├── Cosine Similarity
 └── Logistic Regression / SVM
      │
      ▼
Predicted Category
```

### Workflow

1.  Convert incoming text into an embedding.
2.  Compare with prototype category vectors or feed into a classifier.
3.  Select the category with the highest confidence.
4.  Route the request automatically.

------------------------------------------------------------------------

## Python Example

``` python
import math

def cosine_similarity(a, b):
    dot = sum(x*y for x, y in zip(a, b))
    mag_a = math.sqrt(sum(x*x for x in a))
    mag_b = math.sqrt(sum(y*y for y in b))
    return dot / (mag_a * mag_b)

categories = {
    "TECHNICAL_SUPPORT": [0.86, 0.12, 0.89, 0.05],
    "BILLING_FINANCE": [0.11, 0.88, 0.14, 0.85]
}

ticket = [0.82, 0.15, 0.87, 0.07]

best = max(
    categories,
    key=lambda c: cosine_similarity(ticket, categories[c])
)

print(best)
```
