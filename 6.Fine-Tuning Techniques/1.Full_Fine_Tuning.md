# Full Fine-Tuning

## 1-Liner Definition

**Full fine-tuning** is a training strategy in which every parameter
(weight) of a pre-trained language model is unlocked and updated using a
new, specialized dataset.

------------------------------------------------------------------------

## Short Technical Explanation

In full fine-tuning, the entire neural network is trainable. All
embedding layers, attention layers, feed-forward networks, and output
layers participate in gradient updates.

During training:

1.  The model performs a forward pass.
2.  The loss is computed.
3.  Backpropagation calculates gradients for **every parameter**.
4.  The optimizer updates all weights.

Because gradients and optimizer states must be stored for the entire
model, full fine-tuning requires a large amount of GPU memory (VRAM).

Although it enables deep specialization, it also increases the risk of
**catastrophic forgetting**, where the model loses some of the general
knowledge acquired during pre-training.

------------------------------------------------------------------------

## Real-World Scenarios

### Legal AI Systems

A legal technology company fully fine-tunes a language model on millions
of legal documents so it can understand specialized legal terminology
and document formats.

### Medical Language Models

Healthcare organizations fully fine-tune foundation models on clinical
records and medical literature to improve domain-specific performance.

------------------------------------------------------------------------

## Code Snippet

``` python
import torch
import torch.nn as nn

class CustomModel(nn.Module):

    def __init__(self):
        super().__init__()

        self.layer_1 = nn.Linear(2, 2)
        self.layer_2 = nn.Linear(2, 2)

model = CustomModel()

# Full Fine-Tuning
for name, param in model.named_parameters():

    param.requires_grad = True

    print(
        f"{name}: "
        f"requires_grad={param.requires_grad}"
    )

print("Ready for full fine-tuning.")
```

### Expected Output

``` text
layer_1.weight: requires_grad=True
layer_1.bias: requires_grad=True
layer_2.weight: requires_grad=True
layer_2.bias: requires_grad=True

Ready for full fine-tuning.
```
