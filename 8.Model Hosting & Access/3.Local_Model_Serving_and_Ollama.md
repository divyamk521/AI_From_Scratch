# Local Model Serving and Ollama

## 1-Liner Definition

**Local model serving** is the process of running open-weight AI models
on your own computer or private server so applications can use them
without sending data to external cloud APIs.

------------------------------------------------------------------------

## Short Technical Explanation

A local inference engine runs as a background service and exposes a
local API endpoint (for example, `localhost:11434`).

Applications send prompts to this local endpoint, where the model is
loaded into RAM or GPU memory, performs inference, and returns
responses---all on your own hardware.

**Ollama** simplifies this process by packaging models, runtimes, and a
local API into an easy-to-use tool. It automatically detects available
hardware and serves models through an OpenAI-compatible interface.

The **Ollama Model Library** provides optimized GGUF models, while the
**Ollama SDK** offers Python and JavaScript APIs for interacting with
locally hosted models.

------------------------------------------------------------------------

## Real-World Scenarios

-   Build private AI assistants that never send sensitive data to the
    cloud.
-   Develop and test AI applications offline.
-   Deploy local coding assistants for engineering teams.

------------------------------------------------------------------------

## Common Ollama Commands

``` bash
ollama pull llama3.2
ollama run llama3.2
ollama ls
```

------------------------------------------------------------------------

## Code Snippet

``` python
import ollama

stream = ollama.chat(
    model="llama3.2",
    messages=[
        {"role":"system","content":"You are a helpful assistant."},
        {"role":"user","content":"Explain database indexing."}
    ],
    stream=True
)

for chunk in stream:
    print(chunk["message"]["content"], end="")
```
