# Zero-Shot, One-Shot, and Few-Shot Learning

## 1-Liner Definition

**Zero-shot**, **One-shot**, and **Few-shot** learning are prompting
techniques where an AI model performs a new task after seeing **zero**,
**one**, or **a few** examples of the desired input-output format.

------------------------------------------------------------------------

## Short Technical Explanation

These techniques rely on **In-Context Learning (ICL)**, where the model
learns from examples placed inside the prompt without changing its
trained weights.

### Zero-Shot

The model receives only an instruction and the input.

-   No examples are provided.
-   The model relies entirely on its pre-trained knowledge.

### One-Shot

Exactly **one** example is included before the actual task.

-   Helps establish formatting, tone, or expected output.

### Few-Shot

Typically **3--5 examples** are included.

-   The model identifies patterns from multiple examples using its
    attention mechanism.
-   Usually produces more accurate and consistent outputs for structured
    tasks.

------------------------------------------------------------------------

## Real-World Scenarios

### Automated Data Processing

A backend engineer wants an LLM to convert raw logs into JSON.

-   **Zero-shot:** Output may be inconsistent.
-   **Few-shot:** After seeing several examples, the model reliably
    generates correctly structured JSON.

### Customer Ticket Routing

Customer support tickets are classified into departments.

-   **Zero-shot:** Makes a general prediction.
-   **Few-shot:** Learns routing patterns from example tickets and
    improves classification accuracy.

## Code Snippet

``` python
def generate_prompt(technique: str, user_input: str) -> str:

    if technique == "zero-shot":
        return (
            "Classify the sentiment as Positive or Negative.\n"
            f"Text: {user_input}\n"
            "Sentiment:"
        )

    elif technique == "one-shot":
        return (
            "Classify the sentiment.\n\n"
            "Example:\n"
            "Text: The server response is fast.\n"
            "Sentiment: Positive\n\n"
            "Actual Task:\n"
            f"Text: {user_input}\n"
            "Sentiment:"
        )

    elif technique == "few-shot":
        return (
            "Classify the sentiment.\n\n"
            "Example 1:\n"
            "Text: The server response is fast.\n"
            "Sentiment: Positive\n\n"
            "Example 2:\n"
            "Text: The database connection timed out.\n"
            "Sentiment: Negative\n\n"
            "Example 3:\n"
            "Text: The UI layout is clean and minimal.\n"
            "Sentiment: Positive\n\n"
            "Actual Task:\n"
            f"Text: {user_input}\n"
            "Sentiment:"
        )

live_query = "The API routing throws a 500 error."

print("--- FEW-SHOT PROMPT TEMPLATE ---")
print(generate_prompt("few-shot", live_query))
```
