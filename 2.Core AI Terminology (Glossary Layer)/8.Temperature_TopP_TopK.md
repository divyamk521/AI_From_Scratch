# Temperature, Top-P, and Top-K

## 1-Liner Definition

**Temperature**, **Top-P**, and **Top-K** are inference-time
hyperparameters that control the randomness, creativity, and diversity
of text generated by a Large Language Model (LLM).

------------------------------------------------------------------------

## Short Technical Explanation

During inference, an LLM assigns a probability (logit) to every possible
next token in its vocabulary. Before selecting the next token, these
hyperparameters modify or filter those probabilities.

### Temperature

Temperature scales the logits before converting them into probabilities.

\[ `\text{Scaled Logits}`{=tex} = `\frac{\text{Logits}}{T}`{=tex} \]

-   **Low Temperature (\< 0.3):**
    -   Produces deterministic outputs.
    -   The highest-probability token is almost always selected.
-   **High Temperature (\> 0.8):**
    -   Produces more creative and diverse outputs.
    -   Lower-probability tokens have a greater chance of being
        selected.

------------------------------------------------------------------------

### Top-K Sampling

Top-K limits the candidate pool to the **K highest-probability tokens**.

For example:

-   **Top-K = 50**
    -   Only the 50 most likely tokens are considered.
    -   Every other token is discarded.

------------------------------------------------------------------------

### Top-P (Nucleus Sampling)

Top-P dynamically selects the smallest group of tokens whose cumulative
probability reaches a threshold **P**.

For example:

-   **Top-P = 0.90**
    -   Keep only the most likely tokens whose probabilities add up to
        90%.
    -   Discard the remaining low-probability tail.

------------------------------------------------------------------------

## Real-World Scenarios

### Production Code Generation

For tasks requiring strict formatting and reproducible outputs:

-   Temperature = **0.0**
-   Top-P = **0.1**

This minimizes randomness and improves consistency.

### Creative Writing

For storytelling or brainstorming applications:

-   Temperature = **0.9**
-   Top-P = **0.95**

These settings encourage diverse and creative word choices.

------------------------------------------------------------------------

## Code Snippet

``` python
import numpy as np

# Mock vocabulary and logits
vocab = [
    "code",
    "software",
    "application",
    "banana",
    "spaceship"
]

logits = np.array([4.0, 3.5, 2.0, -1.0, -3.0])

def sample_next_token(
    vocab,
    logits,
    temperature=1.0,
    top_k=3,
    top_p=0.9
):
    # Temperature scaling
    scaled_logits = logits / max(temperature, 1e-6)

    exp_logits = np.exp(scaled_logits)
    probs = exp_logits / np.sum(exp_logits)

    # Top-K filtering
    top_k_indices = np.argsort(probs)[-top_k:]

    filtered_probs = np.zeros_like(probs)
    filtered_probs[top_k_indices] = probs[top_k_indices]
    filtered_probs /= np.sum(filtered_probs)

    # Top-P filtering
    sorted_indices = np.argsort(filtered_probs)[::-1]
    cumulative_probs = np.cumsum(filtered_probs[sorted_indices])

    keep_indices = sorted_indices[cumulative_probs <= top_p]

    if len(keep_indices) == 0:
        keep_indices = [sorted_indices[0]]

    final_probs = np.zeros_like(filtered_probs)
    final_probs[keep_indices] = filtered_probs[keep_indices]
    final_probs /= np.sum(final_probs)

    return np.random.choice(vocab, p=final_probs)

print(
    "Deterministic Output:",
    sample_next_token(vocab, logits, temperature=0.1)
)

print(
    "Creative Output:",
    sample_next_token(vocab, logits, temperature=0.9)
)
```
