# AI Hallucinations

## 1-Liner Definition

An **AI hallucination** is a phenomenon where a Large Language Model
(LLM) confidently generates information that is factually incorrect,
fabricated, misleading, or unsupported by its training data or the
provided context.

------------------------------------------------------------------------

## Short Technical Explanation

Large Language Models are **next-token prediction systems**, not
databases of verified facts.

During inference, an LLM predicts the most probable next token based on
patterns learned during training. It does **not** verify information
against an external source unless connected to retrieval systems such as
RAG or web search.

Hallucinations can occur because of:

-   Incomplete or outdated training data
-   Ambiguous prompts
-   Conflicting information in the training corpus
-   High creativity settings (for example, high temperature)
-   Questions about topics outside the model's knowledge

Since the model is optimized for producing fluent and coherent
language---not guaranteed factual correctness---it may generate answers
that sound convincing but are entirely fabricated.

------------------------------------------------------------------------

## Real-World Scenarios

### Legal Research

An attorney asks an AI to draft a legal brief. The model invents court
cases, fake citations, and nonexistent judicial opinions that appear
authentic but do not actually exist.

### Software Development

A developer asks an LLM to generate code using a library. The model
hallucinates a nonexistent API function or parameter, resulting in
compilation or runtime errors.

------------------------------------------------------------------------

## Code Snippet

``` python
# Mock knowledge base containing verified facts
factual_memory = {
    "who is the author of python": "Guido van Rossum",
    "what is fastapi":
        "A modern, fast web framework for building APIs with Python"
}

def generate_with_hallucination_risk(user_prompt, creativity_factor):
    cleaned_prompt = user_prompt.lower().strip()

    # Return grounded information when available
    if cleaned_prompt in factual_memory and creativity_factor < 0.2:
        return factual_memory[cleaned_prompt]

    # Otherwise generate a fabricated response
    hallucinated_words = [
        "John", "Doe", "invented",
        "it", "in", "2024", "using", "C++"
    ]

    return (
        "Based on statistical text patterns, "
        "the answer is likely: "
        + " ".join(hallucinated_words)
    )

# Grounded response
print(
    generate_with_hallucination_risk(
        "who is the author of python",
        creativity_factor=0.0
    )
)

# Hallucinated response
print(
    generate_with_hallucination_risk(
        "who wrote the custom library xyz",
        creativity_factor=0.8
    )
)
```

### Expected Output

``` text
Guido van Rossum

Based on statistical text patterns, the answer is likely:
John Doe invented it in 2024 using C++
```
