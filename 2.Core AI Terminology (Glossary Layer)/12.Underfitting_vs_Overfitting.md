# Underfitting vs Overfitting

## 1-Liner Definition

**Underfitting** occurs when a model is too simple to learn the
underlying patterns in the data, while **Overfitting** occurs when a
model becomes too complex and memorizes the training data (including
noise), resulting in poor performance on new, unseen data.

------------------------------------------------------------------------

## Short Technical Explanation

### Underfitting (High Bias)

An underfitted model lacks sufficient complexity to capture the
relationship between input features and target values.

Characteristics:

-   Poor performance on the training dataset
-   Poor performance on the test dataset
-   High bias and low variance

Engineers typically reduce underfitting by:

-   Increasing model complexity
-   Adding more informative features
-   Training for more epochs

### Overfitting (High Variance)

An overfitted model has excessive complexity and memorizes the training
data instead of learning general patterns.

Characteristics:

-   Very low training loss
-   High test loss
-   Poor generalization

Common techniques to reduce overfitting include:

-   L1/L2 Regularization
-   Dropout
-   Early Stopping
-   Collecting more training data
-   Data augmentation

------------------------------------------------------------------------

## Real-World Scenarios

### Stock Market Prediction

-   **Underfitting:** A simple linear model cannot capture the nonlinear
    movement of stock prices.
-   **Overfitting:** A highly complex model memorizes every fluctuation
    in historical prices and performs poorly in live trading.

### Email Spam Detection

An overfitted spam filter memorizes exact words, punctuation, and
formatting from training emails, causing it to miss new spam messages
with different wording.

## Code Snippet

``` python
import numpy as np
from sklearn.linear_model import LinearRegression
from sklearn.preprocessing import PolynomialFeatures
from sklearn.pipeline import make_pipeline

# Curved training data
X = np.array([[1], [2], [3], [4]])
y = np.array([2.1, 3.9, 9.2, 15.8])

# Underfitting model (Degree 1)
underfit_model = LinearRegression()
underfit_model.fit(X, y)

# Overfitting model (Degree 5)
overfit_model = make_pipeline(
    PolynomialFeatures(degree=5),
    LinearRegression()
)

overfit_model.fit(X, y)

new_X = [[2.5]]

print(
    "Underfit Prediction:",
    underfit_model.predict(new_X)[0]
)

print(
    "Overfit Prediction:",
    overfit_model.predict(new_X)[0]
)
```
