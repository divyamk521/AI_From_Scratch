# Latency vs Throughput

## 1-Liner Definition

**Latency** is the total time required for a single request to travel
from its source to its destination and receive a response.
**Throughput** is the amount of data or the number of requests a system
can successfully process within a given period of time.

------------------------------------------------------------------------

## Short Technical Explanation

### Latency (Delay)

Latency is measured in units of time such as **milliseconds (ms)** or
**seconds (s)**.

For Large Language Models (LLMs), latency is commonly divided into:

-   **Time-to-First-Token (TTFT):** The time before the model generates
    the first output token.
-   **Inter-Token Latency (ITL):** The time between consecutive
    generated tokens.

Lower latency results in faster responses and a smoother user
experience.

### Throughput (Capacity)

Throughput measures how much work a system can complete over time.

Common metrics include:

-   Requests per Second (RPS)
-   Tokens per Second (TPS)
-   Gigabits per Second (Gbps)

A system may have:

-   **Low latency but low throughput:** Fast for one request but unable
    to handle many simultaneously.
-   **High latency but high throughput:** Slower individual responses
    but capable of processing thousands of requests concurrently.

------------------------------------------------------------------------

## Real-World Scenarios

### Real-Time Voice Assistant

Voice assistants prioritize **low latency**. If the Time-to-First-Token
exceeds about **300 ms**, conversations begin to feel slow and
unnatural.

### Batch Document Processing

When processing thousands of documents overnight, **throughput** becomes
more important than individual request speed. Large batches maximize
overall system efficiency.

## Code Snippet

``` python
import time

# Simulate processing a request
def process_request(request_id):
    time.sleep(0.5)  # Simulated 500 ms latency
    return f"Result {request_id}"

# -----------------------------
# Sequential Processing
# -----------------------------
start_time = time.time()

results_seq = [process_request(i) for i in range(4)]

total_time_seq = time.time() - start_time

print("--- Sequential Execution ---")
print(f"Latency per request: ~0.50 s")
print(f"Total time: {total_time_seq:.2f} s")
print(f"Throughput: {4 / total_time_seq:.2f} requests/s")

# -----------------------------
# Parallel Processing Simulation
# -----------------------------
start_time = time.time()

time.sleep(0.5)  # Simulate four workers finishing together

total_time_parallel = time.time() - start_time + 0.02

print("\n--- Parallel Execution ---")
print(f"Latency per request: ~0.50 s")
print(f"Total time: {total_time_parallel:.2f} s")
print(f"Throughput: {4 / total_time_parallel:.2f} requests/s")
```
