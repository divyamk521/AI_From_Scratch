# Embeddings

## 1-Liner Definition

An **embedding** is a numerical vector generated by an AI model that
captures the semantic meaning, context, and relationships of text,
images, or audio.

------------------------------------------------------------------------

## Short Technical Explanation

Computers cannot directly understand the meaning of words using raw
characters. Embeddings solve this by mapping data into a
**high-dimensional vector space** (commonly 768 to 1536+ dimensions).

An embedding model converts text into a vector (a list of numbers).
Items with similar meanings are positioned close together in this vector
space.

The similarity between two embeddings is commonly measured using
**Cosine Similarity**:

\[ `\text{Similarity}`{=tex} = `\frac{A \cdot B}{\|A\| \|B\|}`{=tex} \]

where:

-   **A · B** = Dot product of the vectors
-   **‖A‖** and **‖B‖** = Magnitudes (lengths) of the vectors

A cosine similarity closer to **1** indicates high semantic similarity.

------------------------------------------------------------------------

## Real-World Scenarios

### Retrieval-Augmented Generation (RAG)

In RAG systems, user queries are converted into embeddings and compared
against document embeddings stored in vector databases (such as
**pgvector**, Qdrant, or Pinecone). Documents with the highest semantic
similarity are retrieved as context for the LLM.

### Recommendation Systems

E-commerce platforms convert user behavior and product descriptions into
embeddings. Products whose vectors are closest to a user's preference
vector are recommended.

------------------------------------------------------------------------

## Code Snippet

``` python
import numpy as np

# Mock semantic vectors
vector_backend  = np.array([0.9, 0.8, 0.1])
vector_software = np.array([0.8, 0.9, 0.2])
vector_banana   = np.array([0.1, 0.1, 0.9])

def calculate_similarity(v1, v2):
    dot_product = np.dot(v1, v2)
    magnitude = np.linalg.norm(v1) * np.linalg.norm(v2)
    return dot_product / magnitude

similarity_tech = calculate_similarity(vector_backend, vector_software)
similarity_fruit = calculate_similarity(vector_backend, vector_banana)

print(f"Similarity ('backend' vs 'software'): {similarity_tech:.4f}")
print(f"Similarity ('backend' vs 'banana'): {similarity_fruit:.4f}")
```

### Expected Output

``` text
Similarity ('backend' vs 'software'): 0.99...
Similarity ('backend' vs 'banana'):   0.3...
```
