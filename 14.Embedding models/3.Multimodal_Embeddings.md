# Multimodal Embeddings

## 1-Liner Definition

**Multimodal embeddings** are dense vectors that represent multiple data
types (text, images, audio, etc.) within a shared vector space, allowing
semantic comparison across different modalities.

------------------------------------------------------------------------

## Short Technical Explanation

Models such as **CLIP** use two encoders:

-   **Text Encoder** (Transformer)
-   **Image Encoder** (Vision Transformer/ResNet)

Both produce embeddings in the same vector space and are trained with
contrastive learning so matching text-image pairs are close together
while mismatched pairs are far apart.

------------------------------------------------------------------------

## Real-World Applications

-   Text-to-image search
-   Zero-shot image classification
-   Product image retrieval
-   Multimodal recommendation systems

------------------------------------------------------------------------

## Architecture

``` text
Text ──► Text Encoder ──┐
                        ├── Shared Embedding Space
Image ─► Image Encoder ─┘
```

------------------------------------------------------------------------

## Code Example

``` python
from transformers import CLIPModel, CLIPProcessor

model = CLIPModel.from_pretrained("openai/clip-vit-base-patch32")
processor = CLIPProcessor.from_pretrained(
    "openai/clip-vit-base-patch32"
)
```
