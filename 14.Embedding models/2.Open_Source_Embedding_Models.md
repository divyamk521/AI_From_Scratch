# Open-Source Embedding Models

## Sentence Transformers

### 1-Liner Definition

**Sentence Transformers** is the standard Python library for running and
fine-tuning open-source embedding models locally with minimal code.

### Technical Explanation

It builds on transformer encoders and applies pooling (typically mean
pooling) to generate a single embedding for an entire sentence. It
integrates tightly with Hugging Face and supports lightweight CPU models
as well as GPU-accelerated models.

``` python
from sentence_transformers import SentenceTransformer, util

model = SentenceTransformer("all-MiniLM-L6-v2")
embeddings = model.encode([
    "Configuring async routing in FastAPI.",
    "Setting up garden trellises."
])
print(util.cos_sim(embeddings[0], embeddings[1]))
```

## BGE Models (BAAI)

### 1-Liner Definition

**BGE (BAAI General Embedding)** models are high-performance open-source
embedding models optimized for semantic and hybrid retrieval.

### Highlights

-   BGE v1.5 series
-   BGE-M3 (Dense + Sparse + Multi-vector)
-   Up to **8192-token** context
-   Excellent multilingual support
-   Ideal for production hybrid RAG

------------------------------------------------------------------------

## E5 Models

### 1-Liner Definition

**E5 models** are Microsoft's embedding models that align search queries
and documents using explicit prefixes.

### Technical Explanation

Use:

-   `query: your search`
-   `passage: indexed document`

This improves asymmetric retrieval where short queries are matched
against long documents.

------------------------------------------------------------------------

## Instructor Models

### 1-Liner Definition

**Instructor models** generate task-aware embeddings by combining an
instruction with the input text.

Example:

``` text
Instruction:
Represent this passage for semantic retrieval.

Text:
FastAPI supports asynchronous request handling.
```

The same model can perform retrieval, classification, clustering, and
similarity simply by changing the instruction.
