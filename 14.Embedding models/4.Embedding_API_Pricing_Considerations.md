# Embedding API Pricing Considerations

## 1-Liner Definition

**Embedding API pricing** involves balancing token processing costs,
embedding model size, indexing expenses, query volume, and vector
database storage to build a cost-efficient semantic search system.

------------------------------------------------------------------------

## Key Pricing Drivers

### 1. Token Usage

Embedding APIs charge **only for input tokens**.

-   **Initial Indexing:** One-time cost to embed existing documents.
-   **Runtime Queries:** Ongoing cost for embedding user search queries.

### 2. Model Tier & Dimensions

Higher-dimensional models provide better semantic quality but cost more
and produce larger vectors.

Examples: - **text-embedding-3-small** -- 1536 dimensions -
**text-embedding-3-large** -- 3072 dimensions

### 3. Batch Processing

Many providers offer **Batch APIs** that process requests asynchronously
at a reduced price, making large-scale indexing significantly cheaper.

### 4. Vector Database Costs

Generated embeddings must be stored in a vector database (e.g.,
pgvector, Milvus, Pinecone). Larger vectors consume more storage, RAM,
and indexing resources.

------------------------------------------------------------------------

## Cost Planning

  Cost Category      Description
  ------------------ -------------------------------------
  Initial Indexing   One-time embedding of documents
  Runtime Queries    Embedding live user searches
  Model Choice       Larger models cost more
  Vector Storage     Database hosting and indexing costs

------------------------------------------------------------------------

## Best Practices

-   Use smaller models for high-volume search.
-   Use batch indexing whenever possible.
-   Choose the lowest embedding dimensionality that meets your accuracy
    requirements.
-   Monitor vector database storage alongside API costs.

------------------------------------------------------------------------

## Code Example

``` python
class EmbeddingCostCalculator:
    def __init__(self, rate):
        self.rate = rate

    def estimate(tokens):
        return (tokens / 1_000_000) * rate

rate = 0.13   # Example price per million tokens
tokens = 50_000_000

print((tokens / 1_000_000) * rate)
```
