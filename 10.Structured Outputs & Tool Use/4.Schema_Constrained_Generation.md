# Schema-Constrained Generation

## 1-Liner Definition

**Schema-constrained generation** is a decoding technique that
mathematically forces an AI model to produce output matching a
predefined schema, such as a JSON Schema or Pydantic model.

------------------------------------------------------------------------

## Short Technical Explanation

Schema-constrained generation uses **grammar-guided decoding**.

When a schema is provided, the inference engine converts it into a
grammar representation. During token generation, any token that would
violate the schema is eliminated, ensuring the final output always
matches the required structure.

This removes the need for post-processing or manual validation.

------------------------------------------------------------------------

## Real-World Scenarios

-   Extract structured business data into databases.
-   Generate consistent JSON for frontend dashboards.
-   Build reliable AI microservices with strict data validation.

------------------------------------------------------------------------

## Code Snippet

``` python
from pydantic import BaseModel
from typing import Literal

class RouteAuditLog(BaseModel):
    endpoint: str
    http_method: Literal["GET", "POST", "PUT", "DELETE"]
    execution_time_ms: int

config = {
    "model": "qwen3-coder-30b",
    "response_format": {
        "type": "json_object",
        "schema": RouteAuditLog.model_json_schema()
    }
}

print(config)
```
