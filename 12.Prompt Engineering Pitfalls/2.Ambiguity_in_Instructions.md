# Ambiguity in Instructions

## 1-Liner Definition

**Ambiguity in instructions** occurs when a prompt contains vague or
open-ended wording, causing an AI model to guess the intended format,
depth, or objective, leading to inconsistent outputs.

------------------------------------------------------------------------

## Short Technical Explanation

LLMs are probabilistic systems. When prompts use unclear phrases such as
**"make it short"**, **"clean this"**, or **"fix the bugs"**, the model
has no precise constraints.

This broadens the possible token predictions, resulting in inconsistent
formatting and varying response quality. Precise prompts with explicit
output formats, limits, and constraints produce more reliable results.

------------------------------------------------------------------------

## Real-World Scenarios

-   A log parser inconsistently identifies "important" errors because
    the prompt never defines what "important" means.
-   A data exporter returns HTML, plain text, or XML instead of the
    required JSON format due to vague instructions.

------------------------------------------------------------------------

## Precision Alignment

  -----------------------------------------------------------------------
  Ambiguous Prompt                      Precise Prompt
  ------------------------------------- ---------------------------------
  Make it short.                        Limit the response to fewer than
                                        50 tokens.

  Fix the bugs.                         Find unawaited async functions
                                        and PostgreSQL connection leaks.

  Format cleanly.                       Return only a JSON array matching
                                        the specified schema.
  -----------------------------------------------------------------------

------------------------------------------------------------------------

## Code Snippet

``` python
import random

class InferenceRouter:

    @staticmethod
    def ambiguous():
        return random.choice([
            "{'status':500}",
            "Server timeout occurred.",
            "ERROR -> 500"
        ])

    @staticmethod
    def precise():
        return '{"error_code":500}'

print(InferenceRouter.ambiguous())
print(InferenceRouter.precise())
```
