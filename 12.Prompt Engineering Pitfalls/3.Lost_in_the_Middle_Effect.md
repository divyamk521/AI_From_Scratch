# Lost-in-the-Middle Effect

## 1-Liner Definition

**The lost-in-the-middle effect** is an LLM limitation where information
at the beginning and end of a long prompt is remembered better than
important details buried in the middle.

------------------------------------------------------------------------

## Short Technical Explanation

Although modern LLMs support very large context windows, attention is
not distributed uniformly.

During inference, the model often gives higher attention to the
beginning and end of the prompt, creating a **U-shaped attention
pattern**. When critical facts appear in the middle of long contexts
(such as RAG documents), they are more likely to be overlooked, leading
to hallucinations or missed answers.

------------------------------------------------------------------------

## Real-World Scenarios

-   A legal AI misses a penalty clause located halfway through a long
    contract.
-   A RAG chatbot ignores the most relevant document because it appears
    in the middle of retrieved context.

------------------------------------------------------------------------

## Engineering Optimizations

-   **Context Reranking:** Move the most relevant documents to the
    beginning.
-   **Delimiters:** Clearly mark important sections with unique
    boundaries.
-   **Instruction Repetition:** Repeat critical instructions near the
    end of the prompt.

------------------------------------------------------------------------

## Code Snippet

``` python
class ContextOrchestrator:

    @staticmethod
    def distribute(docs):
        result = [None] * len(docs)
        left, right = 0, len(docs) - 1

        for i, doc in enumerate(docs):
            if i % 2 == 0:
                result[left] = doc
                left += 1
            else:
                result[right] = doc
                right -= 1

        return result

docs = ["Doc_A", "Doc_B", "Doc_C", "Doc_D", "Doc_E"]
print(ContextOrchestrator.distribute(docs))
```
