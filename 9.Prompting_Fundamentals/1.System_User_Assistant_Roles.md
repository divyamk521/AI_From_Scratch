# System, User, and Assistant Roles

## 1-Liner Definition

**System, user, and assistant roles** are standardized message labels
used by chat-based AI APIs to define instructions, user inputs, and
previous AI responses.

------------------------------------------------------------------------

## Short Technical Explanation

Modern chat APIs exchange conversations as a list of JSON message
objects.

-   **System:** Defines the AI's behavior, rules, and output format.
-   **User:** Contains the current prompt or question.
-   **Assistant:** Stores previous AI responses so the model can
    maintain conversation context.

Since language models are stateless, the complete conversation history
must be sent with each request.

------------------------------------------------------------------------

## Real-World Scenarios

-   Building enterprise chatbots with fixed system instructions.
-   Using few-shot examples by inserting sample user and assistant
    messages.
-   Maintaining multi-turn conversations.

------------------------------------------------------------------------

## Code Snippet

``` python
conversation_history = [
    {"role":"system","content":"You are a helpful coding assistant."},
    {"role":"user","content":"How do I print in Python?"},
    {"role":"assistant","content":"print('Hello World')"},
    {"role":"user","content":"Now print FastAPI Running."}
]

print(len(conversation_history))
```
