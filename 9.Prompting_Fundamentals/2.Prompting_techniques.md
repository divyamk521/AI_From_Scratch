# Prompting Techniques

## API Access vs. Self-Hosting

### 1-Liner Definition

The **API access vs. self-hosting** tradeoff is the choice between using
a managed AI API over the internet or running open-weight models on your
own infrastructure.

------------------------------------------------------------------------

## Short Technical Explanation

### API Access

-   Managed cloud service.
-   No infrastructure management.
-   Pay-per-token pricing.
-   Quick integration.

### Self-Hosting

-   Deploy open models with engines such as **vLLM** or **Ollama**.
-   Complete control over data and models.
-   Requires GPU infrastructure, scaling, and maintenance.

------------------------------------------------------------------------

## Real-World Scenarios

-   **Student Prototype:** Use API access for fast development with
    minimal cost.
-   **Enterprise Legal Assistant:** Self-host models to keep
    confidential documents inside a private network.

------------------------------------------------------------------------

## Tradeoff Comparison

  Factor          API Access           Self-Hosting
  --------------- -------------------- ---------------------------
  Setup           Instant              Days to Weeks
  Data Privacy    Provider-dependent   Full Control
  Pricing         Pay per Token        Fixed Infrastructure Cost
  Maintenance     Low                  High
  Customization   Limited              Extensive

------------------------------------------------------------------------

## Code Snippet

``` python
def calculate_hosting_tradeoff(monthly_chats, tokens_per_chat):
    API_COST_PER_MILLION_TOKENS = 3.0
    GPU_SERVER_COST = 1200.0

    total_tokens = monthly_chats * tokens_per_chat
    api_cost = (total_tokens / 1_000_000) * API_COST_PER_MILLION_TOKENS

    if api_cost > GPU_SERVER_COST:
        return "Recommendation: Self-Host"

    return "Recommendation: API Access"

print(calculate_hosting_tradeoff(50_000, 2_000))
print(calculate_hosting_tradeoff(300_000, 2_000))
```
