# Why Transformers Replaced RNNs

## 1-Liner Definition

**Transformers** replaced **Recurrent Neural Networks (RNNs)** because
they use **Self-Attention** to process entire input sequences in
parallel, overcoming the speed limitations and long-term memory problems
of RNNs.

------------------------------------------------------------------------

## Short Technical Explanation

### Recurrent Neural Networks (RNNs)

RNNs process text **one token at a time**.

To understand the 50th word, the network must first process the previous
49 words. This sequential dependency prevents efficient GPU
parallelization.

RNNs also suffer from the **vanishing gradient problem**, where
information from earlier tokens gradually fades during training, making
it difficult to remember long-range dependencies.

### Transformers

Transformers remove sequential processing by introducing
**Self-Attention**.

Every token attends to every other token simultaneously, allowing:

-   Parallel computation on GPUs
-   Better long-range context understanding
-   Faster training and inference
-   Improved scalability to very long sequences

------------------------------------------------------------------------

## Real-World Scenarios

### Large Codebase Analysis

An AI coding assistant analyzing thousands of lines of code benefits
from Transformers because variables and functions defined early in the
file remain accessible through self-attention.

### Machine Translation

Modern translation systems process entire sentences or documents
simultaneously instead of translating one word at a time, resulting in
faster and more accurate translations.

## Code Snippet

``` python
import numpy as np

# Mock token embeddings
input_tokens = np.random.rand(5, 3)

# -----------------------------
# Sequential RNN Processing
# -----------------------------
def simulate_rnn_processing(tokens):
    hidden_state = np.zeros(3)

    print("RNN Processing:")

    for i, token in enumerate(tokens):
        hidden_state = np.tanh(token + hidden_state)
        print(f"Step {i+1}: Processed sequentially")

    return hidden_state

# -----------------------------
# Parallel Transformer Processing
# -----------------------------
def simulate_transformer_processing(tokens):
    print("\nTransformer Processing:")

    attention_matrix = np.dot(tokens, tokens.T)

    print(
        f"Computed attention matrix with shape "
        f"{attention_matrix.shape}"
    )

    return attention_matrix

rnn_output = simulate_rnn_processing(input_tokens)
transformer_output = simulate_transformer_processing(input_tokens)
```

Step 1: Processed sequentially
Step 2: Processed sequentially

Transformer Processing:
Computed attention matrix with shape (5, 5)
