# Feed-Forward Network (FFN) Sublayer

## 1-Liner Definition

A **Feed-Forward Network (FFN)** is a position-wise neural network
inside every Transformer block that independently processes each token's
contextual representation after the attention mechanism.

------------------------------------------------------------------------

## Short Technical Explanation

A standard Transformer block consists of two major components:

1.  Multi-Head Attention
2.  Feed-Forward Network (FFN)

The attention layer allows tokens to exchange contextual information.
The resulting token representations are then passed to the FFN for
further transformation.

The FFN consists of two linear layers separated by a nonlinear
activation function:

**FFN(x) = max(0, xW₁ + b₁)W₂ + b₂**

where:

-   **W₁** expands the hidden dimension (typically 4× larger).
-   **ReLU** (or modern alternatives like SwiGLU/GELU) introduces
    nonlinearity.
-   **W₂** projects the representation back to the original hidden size.

Unlike attention, the FFN processes **each token independently**. It is
responsible for storing and transforming semantic knowledge encoded in
the model's learned parameters.

Modern architectures such as **Llama**, **Gemma**, and **Mistral**
commonly replace ReLU with **SwiGLU**, which provides improved
performance.

------------------------------------------------------------------------

## Real-World Scenarios

### LLM Knowledge Retrieval

When an LLM answers factual questions, attention identifies relevant
context while the FFN activates internal knowledge stored within the
model's parameters.

### Semantic Search

During vector search pipelines, FFNs transform contextual embeddings
into richer semantic representations that improve retrieval quality.

## Code Snippet

``` python
import torch
import torch.nn as nn
import torch.nn.functional as F

class TransformerFeedForward(nn.Module):

    def __init__(self, d_model, d_ff):
        super().__init__()

        # Expand hidden dimension
        self.w_1 = nn.Linear(d_model, d_ff)

        # Project back
        self.w_2 = nn.Linear(d_ff, d_model)

    def forward(self, x):

        hidden_state = F.relu(
            self.w_1(x)
        )

        output = self.w_2(hidden_state)

        return output

# Input:
# Batch Size = 1
# Sequence Length = 3
# Hidden Dimension = 4

mock_tokens_context = torch.rand(1, 3, 4)

ffn = TransformerFeedForward(
    d_model=4,
    d_ff=16
)

output = ffn(mock_tokens_context)

print("Input Shape :", mock_tokens_context.shape)
print("Output Shape:", output.shape)
```

``` text
Input Shape : torch.Size([1, 3, 4])

(The FFN expands each token representation to a higher-dimensional space and then projects it back to the original hidden dimension.)
```
