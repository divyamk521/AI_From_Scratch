# KV Cache (Key-Value Cache)

## 1-Liner Definition

The **KV Cache (Key-Value Cache)** is an inference optimization
technique that stores the previously computed **Key (K)** and **Value
(V)** vectors for earlier tokens so they do not need to be recomputed
during autoregressive text generation.

------------------------------------------------------------------------

## Short Technical Explanation

Large Language Models generate text one token at a time.

Without a KV Cache, generating the next token requires recomputing the
attention keys and values for every previous token, resulting in
expensive repeated computation.

During inference:

### Prefill Phase

-   The initial prompt is processed.
-   Key and Value vectors for every prompt token are computed.
-   These vectors are stored in GPU memory (VRAM).

### Decoding Phase

For each newly generated token:

1.  Compute its **Query (Q)**, **Key (K)**, and **Value (V)**.
2.  Append the new **K** and **V** vectors to the cache.
3.  Compute attention using the cached history instead of recomputing
    previous tokens.

This reduces decoding complexity from approximately **O(N²)** to
**O(N)** per generated token, greatly improving generation speed.

The tradeoff is increased GPU memory usage because the cache grows with
sequence length.

------------------------------------------------------------------------

## Real-World Scenarios

### Production LLM APIs

Inference engines such as **vLLM** and **TensorRT-LLM** rely heavily on
KV Caches to support thousands of concurrent users while maintaining low
latency.

### Interactive AI Agents

Long conversations and reasoning agents benefit from KV Caching because
generation speed remains fast even as conversation history grows.

## Code Snippet

``` python
import numpy as np

class KVCacheSimulation:

    def __init__(self, feature_dim=4):
        self.k_cache = np.empty((0, feature_dim))
        self.v_cache = np.empty((0, feature_dim))

    def generate(self, token, WQ, WK, WV):

        q = token @ WQ
        k = token @ WK
        v = token @ WV

        self.k_cache = np.vstack([self.k_cache, k])
        self.v_cache = np.vstack([self.v_cache, v])

        scores = q @ self.k_cache.T
        weights = np.exp(scores) / np.sum(np.exp(scores))

        context = weights @ self.v_cache

        return context

WQ = WK = WV = np.eye(4)

cache = KVCacheSimulation()

cache.generate(np.array([0.5,1,0,0.2]), WQ, WK, WV)
context = cache.generate(np.array([0.1,0.8,0.9,0.4]), WQ, WK, WV)

print(context)
```

### Expected Output

``` text
Context vector computed using the cached Key and Value matrices.

The cache grows by one entry after each generated token.
```
