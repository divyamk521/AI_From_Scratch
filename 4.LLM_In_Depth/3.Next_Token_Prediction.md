# Next-Token Prediction

## 1-Liner Definition

The **next-token prediction** objective is the core training strategy of
autoregressive language models, where the model learns to predict the
most probable next token given all previous tokens in a sequence.

------------------------------------------------------------------------

## Short Technical Explanation

Next-token prediction is a form of **Self-Supervised Learning (SSL)**
because the training data provides both the input and the target
automatically.

Given a sequence:

**(x₁, x₂, ..., xₜ)**

the model predicts the probability of the next token:

**P(xₜ₊₁ \| x₁, x₂, ..., xₜ; θ)**

Training minimizes the **Cross-Entropy Loss** between the predicted
probability distribution and the true next token.

During each training step:

1.  Input tokens pass through the Transformer.
2.  The model outputs logits for the next token.
3.  Softmax converts logits into probabilities.
4.  Cross-Entropy Loss is computed.
5.  Backpropagation updates the model parameters.

------------------------------------------------------------------------

## Real-World Scenarios

### Large Language Model Pre-training

Models such as GPT and Llama learn language by repeatedly predicting the
next token across massive text corpora.

### AI Code Completion

Coding assistants predict the next keyword, variable, or function name
while developers type code.

------------------------------------------------------------------------

## Code Snippet

``` python
import torch
import torch.nn as nn
import torch.nn.functional as F

vocab_size = 5

# Model logits
logits = torch.tensor(
    [[1.2, -0.5, 2.1, 4.8, -1.0]],
    requires_grad=True
)

# Correct next token
target = torch.tensor([3])

loss_fn = nn.CrossEntropyLoss()
loss = loss_fn(logits, target)

probabilities = F.softmax(logits, dim=-1)

print("Probabilities:")
print(probabilities.detach().numpy())

print("Loss:", loss.item())

# During training:
# loss.backward()
```
