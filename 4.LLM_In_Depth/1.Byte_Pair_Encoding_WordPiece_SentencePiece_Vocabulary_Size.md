# Byte Pair Encoding (BPE), WordPiece, SentencePiece, and Vocabulary Size Tradeoffs

## 1. Byte Pair Encoding (BPE)

### 1-Liner Definition

**Byte Pair Encoding (BPE)** is a subword tokenization algorithm that
builds its vocabulary by repeatedly merging the most frequently
occurring adjacent token pairs.

### Short Technical Explanation

BPE starts by treating every character as an individual token. It
repeatedly:

1.  Counts all adjacent token pairs.
2.  Finds the most frequent pair.
3.  Merges that pair into a new token.
4.  Adds the new token to the vocabulary.

The process continues until the desired vocabulary size is reached.

During inference, the learned merge rules are applied in the same order
to tokenize unseen text.

### Real-World Scenarios

-   GPT-family and Llama models use byte-level BPE variants.
-   Log compression systems merge repetitive patterns into compact token
    IDs.

### Code Snippet

``` python
from collections import Counter

corpus = ["l o w", "l o w e r", "n e w e s t", "w i d e s t"]

def most_frequent_pair(corpus):
    pairs = Counter()
    for word in corpus:
        chars = word.split()
        for i in range(len(chars)-1):
            pairs[(chars[i], chars[i+1])] += 1
    return pairs.most_common(1)[0][0]

print(most_frequent_pair(corpus))
```

## 2. WordPiece

### 1-Liner Definition

**WordPiece** is a subword tokenizer that merges tokens based on
maximizing language-model likelihood rather than raw frequency.

### Short Technical Explanation

Instead of selecting the most frequent pair, WordPiece chooses merges
that maximize corpus likelihood.

It commonly marks continuation tokens using the **\##** prefix.

Example:

    developing
    → develop + ##ing

### Real-World Scenarios

-   BERT
-   Semantic search
-   Text classification

### Code Snippet

``` python
def tokenize(word, vocab):
    if word in vocab:
        return [word]

    if "develop" in vocab and "##ing" in vocab:
        return ["develop", "##ing"]

    return list(word)

vocab = {"develop","##ing"}
print(tokenize("developing", vocab))
```

## 3. SentencePiece

### 1-Liner Definition

**SentencePiece** is a language-independent tokenizer that treats text
as a raw character stream and explicitly preserves whitespace.

### Short Technical Explanation

SentencePiece performs tokenization directly on raw text without
requiring language-specific preprocessing.

It commonly represents spaces using a special whitespace symbol before
learning BPE or Unigram tokenization.

### Real-World Scenarios

-   T5
-   mT5
-   Multilingual translation models

### Code Snippet

``` python
def sentencepiece(text):
    return text.replace(" ", "▁")

print(sentencepiece("Backend engineer"))
```

## 4. Vocabulary Size Tradeoffs

### 1-Liner Definition

Vocabulary size determines the tradeoff between model size and sequence
length.

### Short Technical Explanation

**Small Vocabulary**

-   Smaller embedding matrix
-   Longer token sequences
-   Lower memory usage

**Large Vocabulary**

-   Larger embedding matrix
-   Shorter token sequences
-   Higher memory usage

Engineers choose vocabulary size based on available compute, memory, and
application requirements.

### Real-World Scenarios

-   Small vocabularies for mobile AI.
-   Large vocabularies for multilingual LLMs.

### Code Snippet

``` python
def embedding_parameters(vocab_size, hidden_dim):
    return vocab_size * hidden_dim

hidden = 4096

print(embedding_parameters(32000, hidden))
print(embedding_parameters(256000, hidden))
```
