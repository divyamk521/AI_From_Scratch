# Context Window (Context Length)

## 1-Liner Definition

A **context window** (or **context length**) is the maximum number of
tokens an AI model can process simultaneously in a single forward pass.

------------------------------------------------------------------------

## Short Technical Explanation

The context window is determined during a model's design and training.

In Transformer models, every token attends to every other token using
**Self-Attention**. The computational complexity grows approximately as
**O(N²)** with respect to the sequence length.

If the input exceeds the maximum context length:

-   Older tokens may be removed using a **First-In, First-Out (FIFO)**
    strategy.
-   Rolling windows or memory optimization techniques may be used.
-   Tokens outside the active context are no longer available to the
    attention mechanism.

Modern models use techniques such as **RoPE (Rotary Position
Embeddings)** and **FlashAttention** to efficiently support much larger
context windows.

------------------------------------------------------------------------

## Real-World Scenarios

### Full Repository Code Review

A developer analyzing an entire codebase requires a large context window
so the model can understand relationships across multiple files without
losing earlier definitions.

### Legal Document Analysis

Comparing two lengthy contracts requires a large context window so
important clauses are not truncated during processing.

## Code Snippet

``` python
class ContextWindowManager:

    def __init__(self, max_context_length):
        self.max_length = max_context_length
        self.token_buffer = []

    def load_input_stream(self, incoming_tokens):

        self.token_buffer.extend(incoming_tokens)

        if len(self.token_buffer) > self.max_length:

            excess = len(self.token_buffer) - self.max_length

            self.token_buffer = self.token_buffer[excess:]

            print(
                f"Context limit reached. "
                f"Removed {excess} old tokens."
            )

    def render_context(self):
        return " ".join(self.token_buffer)

session = ContextWindowManager(6)

session.load_input_stream(
    ["Setup", "FastAPI", "app", "with", "PostgreSQL"]
)

print(session.render_context())

session.load_input_stream(
    ["and", "configure", "Alembic", "migrations"]
)

print(session.render_context())
```

### Expected Output

``` text
Setup FastAPI app with PostgreSQL

Context limit reached. Removed 3 old tokens.

with PostgreSQL and configure Alembic migrations
```
