# Embedding Layer

## 1-Liner Definition

An **embedding layer** is the initial lookup table in a neural network
that converts discrete token IDs into dense vectors of real numbers
representing semantic meaning.

------------------------------------------------------------------------

## Short Technical Explanation

Before a Transformer can process text, token IDs must be converted into
continuous vector representations.

The embedding layer is a learnable matrix:

**E ∈ ℝ\^(V × d_model)**

where:

-   **V** = Vocabulary size
-   **d_model** = Embedding (hidden) dimension

Unlike a standard neural layer, an embedding layer performs an efficient
**index lookup**. If the input token ID is `4`, the model simply
retrieves the 4th row of the embedding matrix.

During training, backpropagation updates these vectors so that tokens
appearing in similar contexts develop similar representations.

------------------------------------------------------------------------

## Real-World Scenarios

### LLM Input Pipeline

Every prompt sent to an LLM is first converted from token IDs into dense
vectors before entering the Transformer.

### Semantic Code Search

Embedding layers convert source code into vectors so developers can
search codebases using natural language instead of exact keyword
matching.

------------------------------------------------------------------------

## Code Snippet

``` python
import torch
import torch.nn as nn

vocab_size = 10
embedding_dim = 4

embedding_layer = nn.Embedding(
    num_embeddings=vocab_size,
    embedding_dim=embedding_dim
)

print("Embedding Matrix")
print(embedding_layer.weight.data)

input_token_ids = torch.tensor([[2, 5, 0]])

dense_vectors = embedding_layer(input_token_ids)

print("Input Token IDs:", input_token_ids.tolist())
print("Output Shape:", dense_vectors.shape)
print(dense_vectors.detach().numpy())
```

### Expected Output

``` text
Embedding Matrix
[[... random weights ...]]

Input Token IDs:
[[2, 5, 0]]

Output Shape:
torch.Size([1, 3, 4])

Each token ID is replaced by its corresponding embedding vector.
```
