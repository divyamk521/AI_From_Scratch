# Knowledge Cutoff Date

## 1-Liner Definition

A **knowledge cutoff date** is the point in time when a model's training
data stops, meaning it has no built-in knowledge of events that occurred
afterward.

------------------------------------------------------------------------

## Short Technical Explanation

During pretraining, a model learns from a massive dataset collected up
to a specific date. After training begins, no new information is added
to the model's parameters.

To answer questions about events after its cutoff, developers typically
use:

-   Retrieval-Augmented Generation (RAG)
-   Web search tools
-   Live databases and APIs

These provide current information at inference time without retraining
the model.

------------------------------------------------------------------------

## Real-World Scenarios

### Software Development

An AI may suggest outdated code if a library changed after its knowledge
cutoff.

### News & Finance

Applications use live market data or web search so the model can discuss
current events accurately.

------------------------------------------------------------------------

## Example Reference

  Model Family       Example Model   Cutoff Behavior
  ------------------ --------------- ------------------------------------
  OpenAI GPT         GPT models      Uses tools for current information
  Anthropic Claude   Claude models   Requires external retrieval
  Google Gemini      Gemini models   Often combines with Google Search

## Code Snippet

``` python
from datetime import datetime

MODEL_CUTOFF = datetime(2026, 1, 1)

def handle_query(event_date):

    if event_date > MODEL_CUTOFF:
        return "Use Web Search or RAG"

    return "Model knowledge is sufficient"

print(handle_query(datetime(2024,6,15)))
print(handle_query(datetime(2026,5,20)))
```
