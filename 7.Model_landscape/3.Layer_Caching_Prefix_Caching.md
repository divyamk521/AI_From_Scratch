# Layer Caching (Prefix Caching / Prompt Caching)

## 1-Liner Definition

**Layer caching (Prefix Caching or Prompt Caching)** is an inference
optimization that stores the KV cache generated from the static prefix
of a prompt so repeated requests can skip recomputing identical tokens.

------------------------------------------------------------------------

## Short Technical Explanation

During the prompt prefill stage, an LLM computes Key-Value (KV)
attention states for each token. If multiple requests begin with the
exact same token sequence, these KV states can be reused instead of
recomputed.

To maximize cache hits:

-   Place static system instructions first.
-   Keep schemas and examples before user input.
-   Append dynamic information (user IDs, timestamps, queries) at the
    end.

Changing the beginning of the prompt invalidates the cache because the
token prefix no longer matches.

------------------------------------------------------------------------

## Real-World Scenarios

### Enterprise Support Bots

Place company policies before the user's question so the policy section
is reused across requests.

### AI Code Review

Keep formatting rules and coding standards as a static prefix while
appending the changing Git diff afterward.

------------------------------------------------------------------------

## Recommended Prompt Layout

``` text
System Instructions
Output Schema
Few-shot Examples
User Input (Dynamic)
```

------------------------------------------------------------------------

## Code Snippet

``` python
import hashlib

cache = {}

def lookup(prefix):
    key = hashlib.md5(prefix.encode()).hexdigest()

    if key in cache:
        return "CACHE HIT"

    cache[key] = True
    return "CACHE MISS"

static_prefix = "You are a FastAPI expert."

print(lookup(static_prefix))
print(lookup(static_prefix))
```
