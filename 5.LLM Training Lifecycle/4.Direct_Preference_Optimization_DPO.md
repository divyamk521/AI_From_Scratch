# Direct Preference Optimization (DPO)

## 1-Liner Definition

**Direct Preference Optimization (DPO)** is an alignment technique that
trains a language model directly on preferred and rejected response
pairs, eliminating the need for a separate Reward Model or Reinforcement
Learning.

------------------------------------------------------------------------

## Short Technical Explanation

Traditional **RLHF** requires two additional stages after supervised
fine-tuning:

1.  Train a Reward Model (RM).
2.  Optimize the language model using PPO.

DPO removes both of these steps.

Instead of learning an explicit reward function, DPO directly compares:

-   Preferred response (winner)
-   Rejected response (loser)

For every prompt, the algorithm increases the probability of generating
the preferred response while decreasing the probability of generating
the rejected one.

This produces behavior similar to RLHF while using a much simpler and
more stable supervised training objective.

------------------------------------------------------------------------

## Real-World Scenarios

### Safer Chatbots

Organizations can train models to consistently choose polite, safe, and
helpful responses over harmful ones using human preference datasets.

### Better Code Assistants

Development teams can teach coding assistants to favor modern frameworks
and best practices by providing preferred and rejected code examples.

------------------------------------------------------------------------

## Code Snippet

``` python
import numpy as np

def calculate_dpo_loss(
    preferred_logprobs,
    rejected_logprobs,
    beta=0.1
):

    difference = preferred_logprobs - rejected_logprobs

    scaled = beta * difference

    loss = -np.log(
        1.0 / (1.0 + np.exp(-scaled))
    )

    return loss

bad_loss = calculate_dpo_loss(
    preferred_logprobs=-3.5,
    rejected_logprobs=-1.2
)

good_loss = calculate_dpo_loss(
    preferred_logprobs=-0.5,
    rejected_logprobs=-4.2
)

print("Bad Alignment:", bad_loss)
print("Good Alignment:", good_loss)
```

### Expected Output

``` text
Bad Alignment Loss : High

Good Alignment Loss : Low

The objective encourages the model to assign higher probability to preferred responses.
```
