# Reward Modeling and Proximal Policy Optimization (PPO)

## 1. Reward Modeling

### 1-Liner Definition

**Reward Modeling** is the process of training a separate neural network
that scores how well an AI model's responses align with human
preferences and instructions.

### Short Technical Explanation

Before Reinforcement Learning from Human Feedback (RLHF), a **Reward
Model (RM)** is trained on human preference data.

For a prompt **x**, humans rank two responses:

-   **y_w** = Preferred (winning) response
-   **y_l** = Rejected (losing) response

The reward model learns to assign a higher score to the preferred
response.

Its objective is based on a pairwise ranking loss:

**L(ψ) = -E\[ log σ(r(x,y_w) − r(x,y_l)) \]**

where:

-   **r(x, y)** = Reward score
-   **σ** = Sigmoid function

After training, the reward model outputs a scalar reward for any
generated response.

### Real-World Scenarios

-   Aligning chatbots to refuse unsafe requests.
-   Teaching customer support models to produce concise, polite
    responses.

### Code Snippet

``` python
import torch
import torch.nn as nn

class RewardModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear = nn.Linear(4, 1)

    def forward(self, x):
        return self.linear(x)

winner = torch.tensor([[0.5,1.2,3.4,0.1]])
loser = torch.tensor([[0.1,-1.0,0.2,2.5]])

model = RewardModel()

rw = model(winner)
rl = model(loser)

loss = -torch.log(torch.sigmoid(rw - rl))

print(loss.item())
```

## 2. Proximal Policy Optimization (PPO)

### 1-Liner Definition

**Proximal Policy Optimization (PPO)** is the reinforcement learning
algorithm that safely updates an LLM using reward scores while
preventing unstable policy changes.

### Short Technical Explanation

During RLHF, the language model acts as the **policy**.

PPO limits how much the updated policy can differ from the previous
policy by clipping the policy ratio.

This stabilizes training, prevents reward hacking, and preserves the
language abilities learned during pre-training and supervised
fine-tuning.

Many RLHF systems also include a **KL-divergence penalty** to keep the
optimized model close to the original SFT model.

### Real-World Scenarios

-   Preventing reward hacking.
-   RLHF training pipelines for instruction-following LLMs.

### Code Snippet

``` python
import numpy as np

def ppo_objective(ratio, advantage, epsilon=0.2):

    unclipped = ratio * advantage

    clipped = np.clip(
        ratio,
        1-epsilon,
        1+epsilon
    ) * advantage

    return min(unclipped, clipped)

print(ppo_objective(1.05, 2.0))
print(ppo_objective(1.40, 2.0))
```
