# Embedding Dimensionality Tradeoffs

## 1-Liner Definition

**Embedding dimensionality tradeoffs** describe the balance between
higher-dimensional embeddings that improve semantic accuracy and
lower-dimensional embeddings that reduce memory, storage, and
computation costs.

------------------------------------------------------------------------

## Short Technical Explanation

The number of embedding dimensions determines how much semantic
information is stored.

-   **High dimensions (1536--3072+)** provide richer semantic
    representations but require more storage, RAM, and computation.
-   **Low dimensions (256--768)** reduce memory usage and improve search
    speed but may lose fine-grained semantic distinctions.

Modern models using **Matryoshka Representation Learning (MRL)**
concentrate the most important information in the early dimensions,
allowing vectors to be truncated with minimal loss in retrieval quality.

------------------------------------------------------------------------

## Real-World Scenarios

-   Reducing vector database costs by truncating embeddings from **1536
    → 512 dimensions**.
-   Deploying lightweight **384-dimensional** embeddings for
    ultra-low-latency search in mobile applications.
-   Scaling enterprise RAG systems while balancing retrieval quality and
    infrastructure costs.

------------------------------------------------------------------------

## Dimensionality Tradeoff Matrix

  Dimensions   Accuracy   Speed      Memory     Best Use Case
  ------------ ---------- ---------- ---------- ---------------------------
  256--384     Lower      Fastest    Lowest     Mobile & edge devices
  768--1536    High       Balanced   Moderate   Production RAG & search
  3072+        Highest    Slowest    Highest    Scientific & legal search

------------------------------------------------------------------------

## Code Example

``` python
import math

def cosine(a, b):
    dot = sum(x*y for x, y in zip(a,b))
    ma = math.sqrt(sum(x*x for x in a))
    mb = math.sqrt(sum(y*y for y in b))
    return dot/(ma*mb)

full_a = [0.85,0.12,0.78,0.05] + [0.01]*1020
full_b = [0.82,0.14,0.75,0.07] + [0.02]*1020

print("1024-D:", cosine(full_a, full_b))
print("4-D:", cosine(full_a[:4], full_b[:4]))
```
