# Semantic Similarity

## 1-Liner Definition

**Semantic similarity** is a metric that measures how closely two pieces
of text are related in meaning by comparing their embedding vectors in a
high-dimensional vector space.

------------------------------------------------------------------------

## Short Technical Explanation

Unlike keyword matching, semantic similarity compares the **meaning** of
text. After an embedding model converts text into vectors, similarity is
computed using mathematical operations such as **Cosine Similarity**,
**Dot Product**, or **Euclidean Distance**.

The most common metric is **Cosine Similarity**:

\[
`\text{Cosine Similarity}`{=tex}=`\frac{\mathbf{A}\cdot\mathbf{B}}{\|\mathbf{A}\|\|\mathbf{B}\|}`{=tex}
\]

-   **1.0** → Nearly identical meaning
-   **0.0** → Unrelated concepts
-   **-1.0** → Opposite directions (rare for modern text embeddings)

------------------------------------------------------------------------

## Real-World Scenarios

-   Grouping similar backend error logs into one bug report.
-   Semantic document search in RAG systems.
-   FAQ matching without exact keyword overlap.
-   Product and content recommendations.

------------------------------------------------------------------------

## Similarity Metrics

  -----------------------------------------------------------------------
  Metric                  Measures                Best Use Case
  ----------------------- ----------------------- -----------------------
  Cosine Similarity       Angle between vectors   Semantic search and RAG

  Dot Product             Angle + magnitude       Fast retrieval with
                                                  normalized embeddings

  Euclidean Distance      Straight-line distance  Standardized vector
                                                  outputs
  -----------------------------------------------------------------------

------------------------------------------------------------------------

## Code Snippet

``` python
import math

def cosine_similarity(v1, v2):
    dot = sum(a*b for a, b in zip(v1, v2))
    mag1 = math.sqrt(sum(a*a for a in v1))
    mag2 = math.sqrt(sum(b*b for b in v2))
    return dot / (mag1 * mag2)

query = [0.88, 0.12, 0.92, 0.03]
doc = [0.85, 0.15, 0.89, 0.05]

print(cosine_similarity(query, doc))
```
