# Dense Embeddings vs Sparse Embeddings

## Dense Embeddings

### 1-Liner Definition

**Dense embeddings** are compact numerical vectors where nearly every
dimension contains a non-zero value, capturing semantic meaning and
contextual relationships.

### Technical Explanation

Dense embeddings are produced by transformer-based embedding models.
They typically contain **768--1536 dimensions**, with almost every
element contributing to the representation.

**Strengths** - Semantic search - Synonym matching - Intent
understanding - RAG retrieval

**Example**

``` text
[0.24, -0.11, 0.89, 0.05, ..., -0.42]
```

------------------------------------------------------------------------

## Sparse Embeddings

### 1-Liner Definition

**Sparse embeddings** are high-dimensional vectors where most values are
**0**, and only a few positions contain non-zero weights representing
exact keywords.

### Technical Explanation

Sparse embeddings are used by models such as **BM25**, **TF-IDF**, and
**SPLADE**. They often contain **30,000--100,000+ dimensions**, with
non-zero entries only for words present in the document.

**Strengths** - Exact keyword matching - Product IDs - Error codes -
Serial numbers

**Example**

``` text
[0.0, 1.45, 0.0, ..., 2.11, 0.0]
```

------------------------------------------------------------------------

## Comparison

  Feature          Dense                     Sparse
  ---------------- ------------------------- ------------------------
  Dimensions       768--1536                 30K--100K+
  Representation   Mostly non-zero           Mostly zeros
  Best At          Semantic understanding    Exact keyword matching
  Weakness         Misses rare identifiers   Misses synonyms

------------------------------------------------------------------------

## Production Recommendation

**Hybrid Search** combines dense and sparse embeddings using ranking
methods such as **Reciprocal Rank Fusion (RRF)** for the best retrieval
performance.

------------------------------------------------------------------------

## Code Example

``` python
def dot(a, b):
    return sum(x*y for x, y in zip(a, b))

dense_q = [0.85, 0.12, 0.90, 0.04]
dense_d = [0.79, 0.15, 0.88, 0.06]

sparse_q = [0.0, 0.0, 2.5, 0.0, 0.0]
sparse_d = [0.0, 0.0, 2.5, 1.1, 0.0]

dense_score = dot(dense_q, dense_d)
sparse_score = dot(sparse_q, sparse_d)

print("Hybrid Score:", dense_score + sparse_score)
```
