# Cosine Similarity, Euclidean Distance & Dot Product

## Cosine Similarity

### 1-Liner Definition

**Cosine Similarity** measures the angle between two vectors, comparing
their semantic direction while ignoring differences in vector length.

### Technical Explanation

Cosine Similarity normalizes vector lengths and is calculated as:

\[
`\text{Cosine Similarity}`{=tex}=`\frac{\mathbf{A}\cdot\mathbf{B}}{\|\mathbf{A}\|\|\mathbf{B}\|}`{=tex}
\]

Typical text embedding scores range from **0.0 (unrelated)** to **1.0
(highly similar)**.

**Best Use Cases** - Semantic search - RAG retrieval - FAQ matching

------------------------------------------------------------------------

## Euclidean Distance

### 1-Liner Definition

**Euclidean Distance** measures the straight-line distance between two
vectors in a high-dimensional space.

### Technical Explanation

\[ `\text{Distance}`{=tex}=`\sqrt{\sum (A_i-B_i)^2}`{=tex} \]

A distance of **0** means identical vectors. Larger values indicate
greater separation.

**Best Use Cases** - Clustering - Fixed-length embeddings -
Classification

------------------------------------------------------------------------

## Dot Product

### 1-Liner Definition

**Dot Product** measures similarity using both vector direction and
magnitude.

### Technical Explanation

\[ `\mathbf{A}`{=tex}`\cdot`{=tex}`\mathbf{B}`{=tex}=`\sum `{=tex}A_iB_i
\]

When embeddings are unit-normalized, Dot Product produces the same
ranking as Cosine Similarity while being computationally faster.

**Best Use Cases** - High-speed vector databases - pgvector - Milvus -
FAISS

## Code Example

``` python
import math

def metrics(a, b):
    dot = sum(x*y for x, y in zip(a,b))
    ma = math.sqrt(sum(x*x for x in a))
    mb = math.sqrt(sum(y*y for y in b))
    cosine = dot/(ma*mb)
    euclidean = math.sqrt(sum((x-y)**2 for x,y in zip(a,b)))
    return dot, cosine, euclidean

v1=[0.90,0.10,0.85,0.05]
v2=[0.88,0.12,0.82,0.06]

print(metrics(v1,v2))
```
