# A/B Testing Prompts

## 1-Liner Definition

**A/B testing prompts** is the practice of comparing two prompt
variations on the same evaluation dataset to determine which produces
the most accurate, reliable, and cost-efficient results.

------------------------------------------------------------------------

## Short Technical Explanation

Rather than changing prompts blindly, developers evaluate two prompt
versions against a **golden dataset**.

Typical evaluation metrics include:

-   **Formatting Success Rate** -- Valid JSON or schema compliance.
-   **Semantic Accuracy** -- Correctness measured by unit tests or an
    LLM judge.
-   **Latency** -- Time to generate a response.
-   **Token Usage** -- Input/output token cost.

The better-performing prompt is promoted to production.

------------------------------------------------------------------------

## Real-World Scenarios

-   Comparing two SQL generation prompts.
-   Evaluating prompt compression without reducing accuracy.
-   Improving structured data extraction pipelines.

## Code Snippet

``` python
class PromptEvaluator:

    def run(self, dataset):
        passed_a = 0
        passed_b = 0

        for case in dataset:
            output_a = "Error code is 500."
            output_b = "500"

            if output_a == case["expected"]:
                passed_a += 1

            if output_b == case["expected"]:
                passed_b += 1

        print("Variant A:", passed_a)
        print("Variant B:", passed_b)

test = [{"expected":"500"}]
PromptEvaluator().run(test)
```
