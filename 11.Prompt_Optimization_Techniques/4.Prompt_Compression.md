# Prompt Compression

## 1-Liner Definition

**Prompt compression** is the process of reducing a prompt's token count
by removing redundant words and unnecessary text while preserving its
meaning and instruction quality.

------------------------------------------------------------------------

## Short Technical Explanation

Large prompts increase API cost and latency. Prompt compression
optimizes prompts before they are sent to an LLM.

Common techniques include:

-   **Heuristic Compression:** Remove extra whitespace, filler words,
    duplicate text, and unnecessary formatting.
-   **Neural Compression:** Use a lightweight model (such as LLMLingua)
    to identify and remove low-information tokens while preserving
    important context.

This typically reduces prompt size by **20--50%**, lowering costs and
improving response speed.

------------------------------------------------------------------------

## Real-World Scenarios

-   Compressing long chatbot conversation histories before sending them
    to an LLM.
-   Reducing large application logs before AI-based debugging.
-   Optimizing enterprise AI pipelines to reduce token usage and
    latency.

## Code Snippet

``` python
import re

class PromptCompressor:

    @staticmethod
    def compress(text):
        text = re.sub(r"\s+", " ", text).strip()

        fillers = ["please", "kindly", "really", "basically"]
        pattern = re.compile(r"\b(" + "|".join(fillers) + r")\b", re.I)
        text = pattern.sub("", text)

        return re.sub(r"\s+", " ", text).strip()

print(PromptCompressor.compress(
    "Please kindly review this code really carefully."
))
```
