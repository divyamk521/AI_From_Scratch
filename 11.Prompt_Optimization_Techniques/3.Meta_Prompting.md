# Meta-Prompting

## 1-Liner Definition

**Meta-prompting** is the technique of asking an AI model to design,
improve, or optimize prompts, allowing it to act as an automated prompt
engineer.

------------------------------------------------------------------------

## Short Technical Explanation

Instead of manually writing a complex prompt, a developer provides a
high-level goal. The model expands it into a production-ready prompt
containing:

-   A clear system role
-   Output constraints
-   Input placeholders
-   Few-shot examples (optional)
-   Negative instructions

The generated prompt can be reused in production systems, multi-agent
workflows, or self-improving AI pipelines.

------------------------------------------------------------------------

## Real-World Scenarios

-   Automatically generating production-quality system prompts for
    developers.
-   Building self-improving customer support bots.
-   Creating AI assistants from simple user descriptions.

------------------------------------------------------------------------

## Meta-Prompt Workflow

``` text
Basic Human Idea
       │
       ▼
Meta-Prompt Engine
       │
       ▼
Production Prompt
 ├── System Role
 ├── Output Constraints
 ├── Input Placeholders
 └── Few-Shot Examples
```

------------------------------------------------------------------------

## Code Snippet

``` python
META_PROMPT = '''
You are an expert Prompt Engineer.

TASK:
{task}

Generate:
- System Role
- Output Constraints
- Input Placeholder
'''

def build_prompt(task):
    return META_PROMPT.format(task=task)

print(build_prompt("Review my FastAPI code"))
```
